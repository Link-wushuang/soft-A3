# 需求分析与用户调研

## 1. 问题背景

当前高校教育面临「千人一面」的困境：传统网课平台和 MOOC 系统向所有学生推送相同的教学资源，无法适应个体差异化的学习需求。具体问题包括：

- **资源同质化**：所有学生看到相同的视频、讲义和练习题，基础薄弱的学生跟不上，基础扎实的学生浪费时间。
- **路径固定化**：学习路径按照教材章节线性排列，无法根据学生的知识掌握情况动态调整。
- **反馈滞后化**：学生做完练习后仅获得对错判定，缺乏针对性的错因分析和补救建议。
- **评估粗粒度**：传统平台只能统计答题正确率，无法刻画学生在具体知识点维度上的掌握程度。

随着大语言模型（LLM）技术的成熟，利用多智能体协作来实现「因材施教」成为可能。

## 2. 目标用户

### 2.1 高校学生（主要用户）

- 需要在有限时间内掌握课程核心知识
- 希望获得与自身水平匹配的学习资源
- 期望在练习后得到即时、具体的反馈
- 典型场景：考前复习、薄弱知识点突破、实验报告准备

### 2.2 高校教师（辅助用户）

- 需要了解班级整体的知识点掌握分布
- 希望管理和维护课程知识库
- 期望减少重复性的个别辅导工作
- 典型场景：查看学情分析、维护知识点和练习题库

## 3. 核心痛点

| 痛点 | 传统平台现状 | 本系统解决方案 |
| --- | --- | --- |
| 学习路径不灵活 | 按教材章节线性推进 | PathPlannerAgent 根据画像动态排序知识点 |
| 资源不个性化 | 统一视频和文档 | 6种资源类型由专属 Agent 根据画像生成 |
| 缺少实时反馈 | 仅显示对错 | EvaluationAgent 给出评分、错因标签和反馈 |
| 画像维度单一 | 仅有成绩和学习时长 | 8维画像：基础水平、学习目标、知识状态、薄弱点、已掌握点、学习偏好、认知风格、时间预算 |
| 资源可靠性无保障 | 依赖人工审核 | VerifierAgent 事实一致性检查 + ContentGuardAgent 安全过滤 |

## 4. 竞品分析

### 4.1 传统网课平台（如超星、智慧树）

- 优势：资源丰富，与高校教务系统集成
- 不足：资源固定，无个性化推荐，不支持动态路径调整
- 差异：本系统根据学生画像实时生成个性化资源，非从固定资源库中筛选

### 4.2 MOOC 平台（如中国大学MOOC、Coursera）

- 优势：名校名师课程，视频质量高
- 不足：学习路径固定，练习反馈机械化，无法根据学生状态调整内容深度
- 差异：本系统支持 LLM 驱动的多维度画像和动态路径规划

### 4.3 AI 辅导产品（如 Khan Academy Khanmigo）

- 优势：对话式交互，一定程度的个性化
- 不足：单一 Agent 模式，资源类型有限，缺乏系统化的课程知识库绑定
- 差异：本系统采用 12+ 个专业 Agent 协作，覆盖从画像到评估的完整学习闭环，且所有生成内容均绑定课程知识库源材料

## 5. 解决方案定位

**EduPath Agent** 定位为「基于大模型的多智能体协作个性化学习系统」，核心理念是：

1. **对话建档**：通过自然语言对话构建 8 维学生画像，而非填写问卷
2. **动态路径**：根据画像和学习反馈动态规划和调整学习路径
3. **多智能体资源生成**：12 个专业 Agent 协作，按需生成 6 种个性化资源
4. **闭环反馈**：练习 → 评估 → 反思 → 画像更新 → 补救资源推送
5. **安全可靠**：源知识绑定生成 + 事实验证 + 内容安全过滤 + 置信度标签

系统以操作系统课程为示范，课程知识库包含 8 章 / 40+ 知识点 / 60+ 练习题 / 8+ 实操案例，可扩展到其他课程。
