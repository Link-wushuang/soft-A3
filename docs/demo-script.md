# 演示脚本（7分钟）

## 第1分钟：问题与系统价值

说明传统课程学习资源“千人一面”的问题：同一讲义无法同时照顾基础薄弱、喜欢图解、时间紧张和偏好实操的学生。引出 EduPath Agent 的画像驱动、多智能体生成和闭环反馈。

## 第2分钟：对话式画像构建

登录学生账号 `demo_student / demo123456`，进入画像页面，输入：

```text
我最近在学操作系统，文件系统这一章比较不会。
尤其是连续分配、链接分配、索引分配的I/O次数计算总是搞混。
我比较喜欢通过图解和例题来学习，希望两天内补一下这部分。
```

展示 8 维画像、置信度、证据和更新原因。

## 第3分钟：个性化学习路径

进入学习路径页面，生成课程路径。重点讲解每个节点的推荐理由：为什么先补文件目录，再学连续分配、链接分配、索引分配和磁盘 I/O。

## 第4分钟：多智能体资源生成

选择“链接分配”或“索引分配”知识点，生成资源。展示：

- Agent trace：检索、讲义、思维导图、练习、案例、拓展阅读、视频分镜、验证、安全检查。
- 6 种资源卡片。
- 安全状态和 warning。

## 第5分钟：练习与反馈

进入练习页面，提交一道错误答案。展示评分、解析、错因标签，以及答题记录如何进入学习效果评估。

## 第6分钟：画像更新与补救

回到画像或资源推荐页面，展示弱点更新、画像日志和补救资源推荐，强调闭环：错题 -> 画像 -> 路径/资源调整。

## 第7分钟：教师端与架构总结

登录教师账号 `demo_teacher / teacher123456`，展示知识点管理和分析页面：

- 薄弱知识点柱状图。
- 资源类型饼图。
- 正确率趋势折线图。

最后用架构图总结多智能体、防幻觉和 Spark 接入方案。
